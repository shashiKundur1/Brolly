# Product

## Register

product

## Users

Developers who run LLM tools and agents daily. Their context: usage-limit anxiety, pricing rug-pulls, and the June 2026 provider shutdown that killed sessions mid-flight. They arrive stressed, mid-task, wanting numbers and a safety net. Judges of the Mesh hackathon are a second audience: they skim for uniqueness, polish, completeness, and a visible Mesh API integration.

## Product Purpose

Brolly is model insurance: a local proxy in front of the Mesh gateway plus a dashboard. It shows live spend/burn/limit projections, routes each request to the cheapest model that passes the user's own mini-benchmark, and hot-swaps a session to the closest-behavior fallback when a model dies. Success: a developer points their tools at Brolly and stops being surprised — by bills or by outages.

## Brand Personality

Warm, wry, plainspoken. A hand-drawn umbrella against 3am-pager weather: the doodle calm is the counterweight to the anxiety the product addresses. Puns about rain are rationed, one per surface.

## Anti-references

- Corporate SaaS dashboards: navy gradients, hero-metric cards, glassmorphism.
- Dark "terminal-cool" minimalism (the Vercel clone look).
- Default-theme shadcn: if a component looks stock, it isn't finished.

## Design Principles

1. The doodle is load-bearing: playfulness delivers calm, never costs legibility.
2. Numbers first, decoration second: the burn rate is the hero, the umbrella watches.
3. One loud coral thing per view: everything else stays sage and paper.
4. Dashed means passive, solid means pressable: the border style is the affordance.
5. Every state is handcrafted: empty, loading, error, and disabled are designed, not defaulted.

## Accessibility & Inclusion

WCAG AA contrast (ink #2f3e36 on all tinted surfaces). Every animation gated behind prefers-reduced-motion. Gochi Hand never below 20px and never on control labels. Full keyboard focus rings (coral) on all interactive components.
